Write an extensive test suite for storypack_importer.py to validate the import process for a storypack directory (e.g., import/BattleChasers). The test should:

Recursively load all files from import/BattleChasers.

Pass them through the storypack_importer pipeline.

Verify that each output file matches the expected structure and data derived from the original input (ensure consistency, no data loss, correct transformations, etc.).

Validate that the correct template was applied during the import process (e.g., check against known templates like chatbot_template.json).

Generate a results report as:

results.json: structured breakdown of test results

results.txt: human-readable summary

results.md: markdown version suitable for GitHub or docs

Include Pytest-compatible tests for CI usage.

Analyze the test results at the end of the test run and:

Print a summary of total tests run, passed, failed.

Flag any inconsistencies or missing data.

Recommend next steps for correcting issues found in the import.

Keep the design modular and clean, ideally in a tests/test_storypack_importer.py file. Feel free to use mock templates or minimal test fixture data where necessary.
