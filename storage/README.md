# 📁 **OpenChronicle Storage Directory**

## **🎯 Overview**
The `storage/` directory contains all OpenChronicle content and data files, organized by purpose and lifecycle. This structure separates user content from system files and provides clear organization for different types of data.

---

## **📂 Directory Structure**

### **🎭 User Content**
- **`storypacks/`** - User story content and narratives
  - Contains complete story packages with characters, scenes, and narrative data
  - Import/export location for story content
  - Primary location for user-created storytelling content

- **`chatbots/`** - AI chatbot configurations  
  - Single-voice AI persona definitions
  - Chatbot personality and behavior configurations
  - Leverages OpenChronicle's CharacterConsistencyEngine

### **💾 Application Data**
- **`data/`** - Active application data and state
  - **`runtime/`** - Current application state and session data
  - **`performance/`** - Metrics, monitoring, and performance reports
  - **`profiles/`** - System profiling and diagnostic data

### **🗄️ System Management**
- **`backups/`** - Permanent backup storage
  - Automated backups of user content
  - System state snapshots
  - Recovery and rollback data

### **🧪 Temporary & Testing**
- **`temp/`** - Temporary files and test artifacts
  - **`test_stories/`** - Generated test story data (40+ test scenarios)
  - **`test_data/`** - Testing artifacts and validation data
  - **`demo/`** - Demonstration content and samples

---

## **🔧 Directory Purposes**

### **User Content Directories**
| Directory | Purpose | Content Type | Persistence |
|-----------|---------|--------------|-------------|
| `storypacks/` | Story content | User narratives, characters, scenes | **Permanent** |
| `chatbots/` | AI personas | Chatbot configurations | **Permanent** |

### **Application Data Directories**
| Directory | Purpose | Content Type | Persistence |
|-----------|---------|--------------|-------------|
| `data/runtime/` | Active state | Session data, current state | **Session** |
| `data/performance/` | Monitoring | Metrics, reports | **Historical** |
| `data/profiles/` | Diagnostics | System profiles | **Historical** |

### **System Management Directories**
| Directory | Purpose | Content Type | Persistence |
|-----------|---------|--------------|-------------|
| `backups/` | Data protection | Automated backups | **Permanent** |
| `temp/` | Temporary data | Test artifacts, demos | **Temporary** |

---

## **📋 Usage Guidelines**

### **✅ Safe to Modify:**
- **`storypacks/`** - Add, edit, or remove your story content
- **`chatbots/`** - Configure AI chatbot personas
- **`temp/`** - Clean up test data (safe to delete contents)

### **⚠️ Application Managed:**
- **`data/runtime/`** - Automatically managed by OpenChronicle
- **`data/performance/`** - Generated by monitoring systems
- **`backups/`** - Automated backup system

### **🚫 Do Not Manually Edit:**
- **`data/profiles/`** - System diagnostic data
- Active runtime state files

---

## **🧹 Maintenance**

### **Regular Cleanup:**
- **`temp/test_stories/`** - Can be cleared periodically (40+ test folders)
- **`temp/test_data/`** - Remove old test artifacts as needed
- **`data/performance/`** - Archive old metrics files

### **Backup Strategy:**
- **User content** (`storypacks/`, `chatbots/`) automatically backed up
- **Application data** (`data/`) backed up selectively
- **Temporary data** (`temp/`) excluded from backups

---

## **🔄 Migration Notes**

### **Path Updates:**
- **Old:** `storypacks/` (root level)
- **New:** `storage/storypacks/` (organized structure)
- Applications should reference the new path structure

### **Import/Export:**
- **Story content** - Use `storage/storypacks/` for import/export operations
- **Chatbot configs** - Use `storage/chatbots/` for AI persona management
- **Templates** - Remain in root-level `templates/` directory

---

## **💡 Directory Philosophy**

**🎯 ORGANIZED:** Clear separation by purpose and lifecycle  
**🔐 PROTECTED:** User content isolated from system files  
**🧹 MAINTAINABLE:** Easy cleanup and maintenance procedures  
**📈 SCALABLE:** Structure supports growth and expansion  

This storage organization provides a clean, maintainable structure that separates user content from system data while maintaining easy access to all OpenChronicle functionality!
